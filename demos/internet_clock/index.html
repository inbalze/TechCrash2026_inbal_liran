Internet Clock

ESP32 fetches real time from an NTP server over WiFi and sends it to the FPGA, which displays hours:minutes:seconds on the 7-segment displays.

Block Diagram

How It Works

  1. ESP32 connects to WiFi hotspot and syncs with an NTP time server
  2. Every second, ESP32 sends the current time as ASCII "HH:MM:SS\r\n" over UART2 (GPIO16)
  3. FPGA receives the serial stream on GPIO[0] at 9600 baud
  4. An ASCII parser extracts the 6 digit values from the time string
  5. On newline, the parsed digits commit to the 7-segment display drivers
  6. HEX5:HEX4 = hours, HEX3:HEX2 = minutes, HEX1:HEX0 = seconds

Wiring

ESP32 PinWireDE10-Lite PinFunction
GPIO16→Arduino IO0 (GPIO[0])UART TX → RX
GND—Arduino GNDCommon ground

Note: Only 2 wires needed for this demo (TX + GND). The Arduino header has male pins, so use simple male-male jumper wires. Both boards run at 3.3V logic, no level shifter required.

Reset

Flip SW[9] down (0) then up (1) to reset the FPGA parser. All displays will blank until the next valid time string arrives.

Project Files

esp32/src/main.cpp

WiFi + NTP client, sends time string over UART2 every second

fpga/src/internet_clock_top.sv

Top module: UART RX, ASCII parser, 7-segment commit logic

fpga/src/uart_rx.sv

Parameterized UART receiver (50 MHz, 9600 baud, 8N1)

fpga/src/seven_segment.sv

BCD to active-low 7-segment decoder with blank input

esp32/platformio.ini

PlatformIO config (ESP32, Arduino framework)

fpga/internet_clock_top.qsf

Quartus pin assignments for DE10-Lite

Build & Flash

FPGA

cd demos/internet_clock/fpga quartus_sh --flow compile internet_clock_top quartus_pgm -m jtag -o "p;output_files/internet_clock_top.sof"

ESP32

cd demos/internet_clock/esp32 # Set proxy if behind Intel firewall: set HTTP_PROXY=http://proxy-iil.intel.com:912 set HTTPS_PROXY=http://proxy-iil.intel.com:912 pio run -t upload

Expected Behavior

  • After power-on, 7-segment displays are blank (waiting for first valid time)
  • ESP32 serial monitor shows WiFi connection, NTP sync, then time every second
  • Once synced, FPGA displays update every second showing Israel local time
  • Time survives WiFi disconnects (ESP32 internal RTC keeps counting)

Customization Ideas

  • Change GMT_OFFSET and DST_OFFSET in main.cpp for your timezone
  • Add decimal points on HEX4 and HEX2 as colon separators (set seg[7]=0)
  • Use SW[0] to toggle 12h/24h format
  • Add alarm functionality with KEY[0] and the buzzer
