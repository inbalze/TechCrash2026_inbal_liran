FPGA Board

Intel DE10-Lite

A powerful yet accessible FPGA development board built around the Intel MAX 10 device. Perfect for digital design — from simple LED control to complete CPUs.

FPGAIntel MAX 10 (10M50DAF484C7G) — 50K logic elements
Clock50 MHz onboard oscillator
Switches10× slide switches (active-high)
Buttons2× push-buttons (active-low, Schmitt trigger)
LEDs10× red LEDs
7-Segment6× displays (active-low, includes decimal point)
VGA4-bit per channel (R, G, B) — 640×480 @ 60Hz
Arduino Header16 I/O pins — connects to ESP32 and addon peripherals
GPIO36 general-purpose I/O pins
ADC6 analog channels (12-bit, onboard MAX 10 ADC)
SDRAM64 MB (16-bit bus)
AccelerometerOnboard ADXL345 via SPI
ProgrammingUSB-Blaster via USB cable (JTAG)

Development Tools

  • Intel Quartus Prime Lite 17.1 — FPGA design, compilation, and programming
  • ModelSim-Altera — simulation and waveform analysis
  • SystemVerilog / Verilog — hardware description language
Controller Kit

ESP32 Controller Kit

A breadboard-based kit built around the ESP32-WROOM-32 DevKit, featuring an OLED display, a push button, and a passive buzzer. Simple, hackable, and perfect for rapid prototyping.

MCUESP32-WROOM-32 (dual-core Xtensa LX6, 240 MHz)
BoardESP32 DevKit V1, 30-pin
Flash4 MB
WiFi802.11 b/g/n (2.4 GHz)
BluetoothBLE 4.2 + Classic
DisplaySSD1306 OLED 128×64 (I2C, address 0x3C)

BuzzerPassive buzzer on GPIO 23 (PWM tone output)

Development Tools

  • VS Code + PlatformIO — build, flash, and debug
  • Arduino framework — familiar C/C++ programming model
  • GitHub Copilot — AI-assisted firmware development

How They Connect

The two boards communicate through wired interfaces — this is the core engineering challenge.

UART (Serial)

TX/RX serial communication. Simple, proven, bidirectional. Connect FPGA GPIO pins to ESP32 UART pins.

SPI

High-speed synchronous bus. FPGA as master or slave, ESP32 on the other end. Great for fast data transfer.

I2C

Two-wire bus with addressing. Good for multi-device setups and sensor-like data exchange.

GPIO Direct

Simple parallel wires — bit-bang your own protocol. Maximum flexibility, minimum abstraction.

Get Started

Everything you need to go from unboxing to building. Follow the steps, explore on your own, and ask your AI copilot for help.

Your Setup Checklist

Complete these steps in order. Each one builds on the previous.

1

Pick Up Your Hardware Kit

Collect your DE10-Lite FPGA board and ESP32 controller kit (with OLED, button, and buzzer). Check that you have USB cables, jumper wires, and a breadboard.

2

Install Quartus Prime Lite 17.1 (FPGA)

Download and install Intel Quartus Prime Lite 17.1 with ModelSim and MAX 10 device support. Install the USB-Blaster driver so you can program the DE10-Lite.

  • Download: Quartus Prime Lite 17.1 for Windows
  • Select: Quartus + ModelSim + MAX 10 device support
  • Install to: C:\intelFPGA_lite\17.1\
  • USB-Blaster driver: C:\intelFPGA_lite\17.1\quartus\drivers\usb-blaster

See the full guide in the de10lite-board-and-build skill

3

Install VS Code + PlatformIO (ESP32)

Install Visual Studio Code, then add the PlatformIO extension. PlatformIO automatically downloads the ESP32 toolchain, compiler, and flasher.

  • Download VS Code: code.visualstudio.com
  • Install extension: search "PlatformIO IDE" in Extensions
  • Install USB driver: CP2102 or CH340 (check Device Manager)

See the full guide in the esp32-firmware skill

4

Set Up GitHub Copilot

GitHub Copilot is your AI engineering teammate. Install the extension in VS Code and sign in with your GitHub account. Use it for SystemVerilog, C/C++, and everything in between.

  • Install: search "GitHub Copilot" in VS Code Extensions
  • Sign in with your GitHub account
  • Free for students: GitHub Student Developer Pack
5

Test Your FPGA — LED Smoke Test

Create a minimal Quartus project that maps the 10 slide switches to the 10 LEDs. Compile, program the board, and flip switches. LEDs light up = your FPGA toolchain works.

See the step-by-step project files in the de10lite-board-and-build skill → "Quick Smoke Test"

6

Test Your ESP32 — OLED + Button + Buzzer

Create a PlatformIO project that displays "Hello" on the OLED, detects button presses, and beeps the buzzer. Build, flash, and verify all three peripherals work.

See the complete smoke test code in the esp32-firmware skill → "Smoke Test"

7

Connect FPGA ↔ ESP32

Wire the two boards together using jumper cables (UART, SPI, I2C, or direct GPIO). Send data from one to the other. This is the core skill for every challenge.

Tip: Start simple: UART TX from FPGA GPIO → ESP32 RX pin. One wire + ground.

8

Try a Practice Challenge

Head to the Practice Challenges tab and take on the Internet Clock, Retro Arcade, or Volt Meter. No solutions provided — figure it out.

Medium

Internet Clock

ESP32 + FPGA + WiFi — real-time clock on seven-segment displays

Build a real-time clock powered by the internet. The ESP32 connects to a public time API over WiFi, retrieves the current time, and continuously transmits it to the DE10-Lite FPGA. The FPGA displays the live hours, minutes, and seconds on its six seven-segment displays.

Requirements

  • ESP32 fetches the current time from a web API (e.g., WorldTimeAPI, NTP server)
  • ESP32 transmits time data to the FPGA over a wired interface (UART, SPI, or I2C via GPIO/Arduino header)
  • FPGA displays the time on HEX5-HEX0 in HH.MM.SS format
  • Clock updates in real time (at least once per second)
  • System works autonomously after power-on (no PC required after programming)

What You'll Learn

FPGA ↔ ESP32 Communication WiFi & HTTP/NTP Serial Data on FPGA Seven-Segment Driving
Hard

Retro Arcade

FPGA Controls → ESP32 Game Engine — play on the OLED screen

Create a playable retro-style game that runs on the ESP32 kit's OLED screen, where the game controls (movement, actions) come from the DE10-Lite FPGA's buttons and switches. The FPGA reads the physical inputs and sends control commands to the ESP32, which renders the game.

Requirements

  • Game runs on the ESP32 kit's OLED screen (any retro game: Pong, Snake, Breakout, Space Invaders, etc.)
  • Player controls are physically on the FPGA: buttons (KEY[0], KEY[1]), switches, or addon joystick
  • FPGA reads inputs and transmits control commands to ESP32 over a wired interface
  • Game must be playable — responsive controls, visible graphics, win/lose condition
  • Bonus points for creativity, polish, and fun factor

What You'll Learn

FPGA → ESP32 Communication Button & Switch Input Game Loop & Graphics Real-Time Input Handling
Medium

Volt Meter

FPGA ADC → ESP32 Display — measure and show real voltage

Turn your hardware kit into a digital volt meter. The DE10-Lite FPGA uses its onboard ADC to probe an external voltage source (e.g., a battery, a potentiometer divider, or a power supply output). It measures the voltage and sends the reading to the ESP32, which displays the voltage value on its OLED screen.

Requirements

  • FPGA reads voltage through its onboard ADC (MAX 10 ADC, channels available via GPIO/Arduino header)
  • Measured voltage is transmitted from FPGA to ESP32 over a wired interface
  • ESP32 displays the voltage on its OLED screen (e.g., "3.27 V") with at least 2 decimal places
  • Display updates in real time (at least 2 updates per second)
  • Measurement range: 0-3.3 V (the ADC reference voltage)

Safety

Do NOT connect voltages above 3.3 V to the ADC input. Doing so will permanently damage the FPGA.

What You'll Learn

MAX 10 Onboard ADC Analog Measurement FPGA → ESP32 Data TX Formatted Numeric Display

GitHub Copilot

Your AI pair programmer, built into VS Code.

Copilot Chat

Open the chat panel (Ctrl+Shift+I) and ask anything: "How do I create a UART receiver in SystemVerilog?", "Explain this error", "Generate a PlatformIO project for ESP32 with OLED".

Inline Completions

Start typing code and Copilot suggests the rest. Write a comment describing what you want, press Tab, and watch it appear. Works with SystemVerilog, C, C++, and more.

Copilot Edits

Select code, press Ctrl+I, and tell Copilot what to change: "Add a reset signal", "Convert this to a state machine", "Fix the timing". It edits your code in-place.

Agent Mode

Give Copilot a high-level goal and let it work autonomously — it can read files, run terminal commands, search your codebase, and iterate until the task is done.

Pro Tips for Effective Use

  • Be specific: "Generate a SystemVerilog module that receives UART at 115200 baud on GPIO pin 12 and outputs 8-bit data" → better results than "make a UART".
  • Provide context: Open relevant files in your editor. Copilot reads your open tabs to understand your project.
  • Iterate: First answer not perfect? Say "make it synchronous" or "add error handling". Copilot refines.
  • Attach files: In Chat, use @workspace to let Copilot search your entire project, or drag files into the chat.
  • Ask it to explain: Don't just generate — ask "explain this code line by line" to actually learn.
Install: Search "GitHub Copilot" in VS Code Extensions → Install → Sign in with GitHub.

What Are Skills?

A Copilot Skill is a Markdown file (SKILL.md) containing structured domain knowledge -- pin mappings, code patterns, build commands, and design recipes. When placed in your workspace, Copilot reads it and becomes an expert on that topic.

How to Use Them
